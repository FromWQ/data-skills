# 数仓健康度评估全流程链路文档

> 本文档详细梳理数仓健康度评估体系的完整链路，包括 skill_0（总控）串联 skill_1/2/3 执行专项评估，最终生成综合健康报告。

---

## 一、整体架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                        数仓健康度评估体系                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  skill_0 (总控) ────┐                                              │
│                     │                                               │
│       ┌─────────────┼─────────────┐                                │
│       │             │             │                                │
│       ▼             ▼             ▼                                │
│  skill_1        skill_2       skill_3                              │
│  模型规范检查    代码规范检查    数据质量检查                        │
│       │             │             │                                │
│       ▼             ▼             ▼                                │
│  model_check   code_check    quality_check                         │
│  _report.md    _report.md    _report.md                            │
│       │             │             │                                │
│       └─────────────┼─────────────┘                                │
│                     ▼                                               │
│            ┌─────────────────┐                                     │
│            │  dw_healthy_    │                                     │
│            │   report.md     │  (综合健康报告)                      │
│            └─────────────────┘                                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.1 专项评估说明

| 专项 | Skill 文件 | 检查内容 | 输出报告 |
|------|-----------|---------|---------|
| 模型规范检查 | skill_1_model_check | 表命名、字段规范、分区规范、存储规范 | `model_check_report.md` |
| 代码规范检查 | skill_2_code_check | SQL 安全、性能、分层、代码规范 | `code_check_report.md` |
| 数据质量检查 | skill_3_quality_check | 完整性、准确性、一致性、及时性 | `quality_check_report.md` |

### 1.2 综合评分公式

```
综合得分 = 模型规范×25% + 代码质量×25% + 数据质量×30%

权重说明：
- 模型规范 (25%)：表设计是基础
- 代码质量 (25%)：SQL 质量影响运行效率
- 数据质量 (30%)：数据准确性最重要
```

## 二、各专项评估链路详情

### 2.1 模型规范检查 (skill_1)

#### 2.1.1 输入

| 输入类型 | 说明 | 位置/格式 |
|---------|------|----------|
| SQL 文件 | 建表 DDL 语句 | `input/*.sql` |
| 词根文件 | 表名词根对照表 | `input/项目词根.xlsx` |

#### 2.1.2 执行步骤

```
步骤 1: 读取 SQL 文件
    │
    ▼
步骤 2: 识别表分层
    │   ods_ → ODS 层
    │   dwd_ → DWD 层
    │   dws_ → DWS 层
    │   dim_ → DIM 层
    │   ads_ → ADS 层
    │
    ▼
步骤 3: 执行 4 大类检查
    │   ├─ 表命名规范（前缀、长度、命名规则）
    │   ├─ 字段规范（注释、命名、类型）
    │   ├─ 分区规范（分区字段设计）
    │   └─ 存储规范（格式、压缩配置）
    │
    ▼
步骤 4: 计算得分
    │   得分 = 100 - 各项扣分
    │
    ▼
步骤 5: 生成报告
        ├─ model_check_report.md
        └─ model_check_result.xlsx
```

#### 2.1.3 检查规则详情

| 检查项 | 检查点 | 扣分标准 |
|--------|--------|---------|
| 表命名规范 | 分层前缀、长度、命名规则 | 5-10 分 |
| 字段规范 | 注释完整性、命名规范、类型统一 | 2-5 分/字段 |
| 分区规范 | 分区字段设计、分区策略 | 5-10 分 |
| 存储规范 | 存储格式、压缩配置 | 5-10 分 |

#### 2.1.4 产出

| 产出类型 | 说明 | 位置 |
|---------|------|------|
| Markdown 报告 | 详细检查报告 | `model_check/output/model_check_report.md` |
| Excel 结果表 | 检查结果明细 | `model_check/output/model_check_result.xlsx` |

### 2.2 代码规范检查 (skill_2)

#### 2.2.1 输入

| 输入类型 | 说明 | 来源 |
|---------|------|------|
| SQL 代码 | 待检查的 SQL 语句 | `input/task_info_data.xlsx` |
| 检查规则 | 45 条检查规则 | `references/code_rules.md` |

#### 2.2.2 执行步骤

```
步骤 1: 读取 Excel 中的 SQL 代码
    │
    ▼
步骤 2: 识别代码所属分层
    │   ├─ 根据表名识别：ods_/dwd_/dws_/dim_/ads_
    │   └─ 根据任务类型识别
    │
    ▼
步骤 3: 执行 5 大类规则检查
    │   │
    │   ├─ 安全合规检查（敏感数据、SQL 注入、权限）
    │   ├─ 性能优化检查（全表扫描、笛卡尔积、分区剪裁）
    │   ├─ 分层原则检查（ODS/DWD/DWS/ADS 规范）
    │   ├─ 代码规范检查（命名、注释、格式）
    │   └─ SQL 反模式检查（SELECT *、隐式转换、负向查询）
    │
    ▼
步骤 4: 计算评分
    │   得分 = 100 - (BLOCK×25) - (WARN×10) - (INFO×3)
    │
    ▼
步骤 5: 生成报告
        ├─ code_check_report.md
        └─ code_check_result.xlsx
```

#### 2.2.3 规则分类详情

| 规则类型 | 规则数量 | 严重程度 | 扣分权重 |
|---------|---------|---------|---------|
| 安全合规检查 | 6 条 | BLOCK | 25 分 |
| 性能优化检查 | 6 条 | BLOCK/WARN | 25/10 分 |
| 分层原则检查 | 4 条 | WARN | 10 分 |
| 代码规范检查 | 22 条 | WARN/INFO | 10/3 分 |
| SQL 反模式检查 | 6 条 | INFO | 3 分 |

#### 2.2.4 产出

| 产出类型 | 说明 | 位置 |
|---------|------|------|
| Markdown 报告 | 详细检查报告 | `code_check/output/code_check_report.md` |
| Excel 结果表 | 每条 SQL 的检查结果 | `code_check/output/code_check_result.xlsx` |

### 2.3 数据质量检查 (skill_3)

#### 2.3.1 前置依赖

| 依赖项 | 说明 | 位置 |
|--------|------|------|
| db_config.json | 数据库连接配置 | `quality_check/input/` |
| db_connector.py | 数据库连接工具 | `quality_check/py_scripts/` |
| meta_table_column 表 | 存储表字段元数据 | 数据库 |
| dq_check_result 表 | 存储校验结果 | 数据库 |
| rule_v2.md | 质量校验规则集 | `quality_check/references/` |

#### 2.3.2 执行步骤

```
步骤 0: 初始化环境
    │   ├─ 创建 meta_table_column 表
    │   ├─ 创建 dq_check_result 表
    │   └─ 同步元数据
    │
    ▼
步骤 1: 同步元数据
    │   ├─ 连接数仓数据库
    │   ├─ 扫描以 ods/dwd/dws/dim/ads 开头的表
    │   ├─ 提取字段信息（名称、类型、注释）
    │   └─ 写入 meta_table_column 表
    │
    ▼
步骤 2: 生成校验 SQL
    │   │
    │   ├─ 通用规则 (COM_001~COM_007)
    │   │   • 分区连续无缺失
    │   │   • 数据量波动合理
    │   │   • 关键字段非空率
    │   │   • 时间字段无异常
    │   │   • 脏数据/乱码检查
    │   │   • 金额非负检查
    │   │   • 枚举值合法性
    │   │
    │   └─ 分层专属规则 (ODS/DWD/DWS/DIM/ADS)
    │
    ▼
步骤 3: 执行校验 SQL
    │   ├─ 逐条执行校验 SQL
    │   ├─ 记录执行结果
    │   └─ 写入 dq_check_result 表
    │
    ▼
步骤 4: 生成评估报告
        ├─ quality_check_report.md
        └─ quality_check_result.xlsx
```

#### 2.3.3 评分公式

**单表评分：**
```
单表得分 = 100 - (BLOCK 失败数×25) - (WARN 失败数×10) - (INFO 失败数×3)
```

**分层评分：**
```
分层得分 = Σ(该分层所有表得分) / 该分层表数量
```

**整体评分：**
```
整体得分 = Σ(各分层得分 × 权重)
权重：ODS 15%, DWD 25%, DWS 25%, DIM 20%, ADS 15%
```

#### 2.3.4 产出

| 产出类型 | 说明 | 位置 |
|---------|------|------|
| Markdown 报告 | 详细评估报告 | `quality_check/output/quality_check_report.md` |
| Excel 结果表 | 每条规则的校验结果 | `quality_check/output/quality_check_result.xlsx` |

## 三、综合评估链路

### 3.1 整体执行流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    数仓健康度评估完整流程                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  【阶段一：准备工作】                                             │
│    1. 确认环境依赖 (Python 3.8+, pip 依赖包)                      │
│    2. 确认配置文件 (db_config.json)                              │
│    3. 准备待检查数据                                             │
│       • skill_1: input/*.sql                                    │
│       • skill_2: input/task_info_data.xlsx                      │
│       • skill_3: 数仓数据库                                      │
│                                                                  │
│  【阶段二：执行专项评估】                                         │
│                                                                  │
│    skill_1(模型规范) ──▶ 4 步骤 ──▶ model_check_report.md       │
│                                                                  │
│    skill_2(代码检查) ──▶ 6 步骤 ──▶ code_check_report.md        │
│                                                                  │
│    skill_3(数据质量) ──▶ 5 步骤 ──▶ quality_check_report.md     │
│                                                                  │
│  【阶段三：生成综合报告】                                         │
│    1. 解析三个专项报告                                           │
│    2. 提取各专项评分                                             │
│    3. 计算综合得分                                               │
│    4. 生成 dw_healthy_report.md                                  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 执行命令

```bash
# 方式一：执行完整流程
python Data-Governance/py_scripts/dw_healthy_checker.py --all

# 方式二：分步执行各专项
python Data-Governance/model_check/py_scripts/model_checker.py
python Data-Governance/code_check/py_scripts/code_checker.py --all
python Data-Governance/quality_check/py_scripts/dq_quality_checker.py --all

# 方式三：仅生成综合报告（需先执行专项）
python Data-Governance/py_scripts/dw_healthy_checker.py --report
```

### 3.3 专项输入输出汇总

| 专项 | 输入 | 输出报告 | 评分提取 |
|-----|------|---------|---------|
| skill_1 模型规范 | SQL 文件、词根表 | `model_check_report.md` | 整体平均分 |
| skill_2 代码检查 | task_info_data.xlsx | `code_check_report.md` | 分层平均分 |
| skill_3 数据质量 | 数仓数据库 | `quality_check_report.md` | 整体健康评分 |

### 3.4 综合评分计算

**评分公式：**
```
综合得分 = (模型规范评分×0.25 + 代码质量评分×0.25 + 数据质量评分×0.30) / 0.8
```

**评分等级：**

| 分数范围 | 等级 | 说明 |
|---------|------|------|
| 90-100 | 优秀 | 数仓健康状态优秀，可放心使用 |
| 75-89 | 良好 | 数仓健康状态良好，存在轻微问题 |
| 60-74 | 一般 | 数仓健康状态一般，需要关注修复 |
| 40-59 | 较差 | 数仓健康状态较差，需要尽快修复 |
| 0-39 | 危险 | 数仓健康状态危险，必须立即处理 |

```markdown

- 综合评分详情
```

## 七、常见问题

### 7.1 数据库连接失败

**错误信息：**
```
pymysql.err.OperationalError: Can't connect to MySQL server
```

**解决方案：**
1. 检查 `db_config.json` 配置是否正确
2. 确认数据库服务已启动
3. 检查网络连接和防火墙

### 7.2 依赖库缺失

**错误信息：**
```
ModuleNotFoundError: No module named 'openpyxl'
```

**解决方案：**
```bash
pip3 install pandas openpyxl pymysql
```

### 7.3 报告生成失败

**错误信息：**
```
FileNotFoundError: [Errno 2] No such file or directory
```

**解决方案：**
1. 确认专项报告已生成
2. 检查输出目录是否存在
3. 检查文件权限

## 八、最佳实践

1. **定期执行**: 建议每周或每月执行一次完整健康检查
2. **优先修复**: 优先处理 BLOCK 级别问题
3. **持续监控**: 建立质量监控机制，及时发现问题
4. **文档沉淀**: 将常见问题和解决方案沉淀为文档
5. **自动化**: 将健康检查集成到 CI/CD 流程中
