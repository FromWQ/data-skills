
## 三、部署步骤

### 步骤 1：复制文件

将整个 `skill_3_quality_check` 文件夹复制到目标电脑的任意位置：

```bash
# 方式 1：使用压缩包
tar -czf skill_3_quality_check.tar.gz ~/skill_3_quality_check/
# 传输到目标电脑后解压到任意位置
tar -xzf skill_3_quality_check.tar.gz -C ~/Documents/

# 方式 2：使用 git
git clone <repository-url> ~/skill_3_quality_check

# 方式 3：直接拷贝
cp -r /source/skill_3_quality_check ~/Documents/
```

### 步骤 2：检查 Python 版本

```bash
python3 --version
# 应显示 Python 3.8.x 或更高版本
```

### 步骤 3：安装依赖

```bash
pip3 install pymysql
```

### 步骤 4：配置数据库连接

编辑 `input/db_config.json`：

```bash
# 进入项目目录（根据你的实际位置修改）
cd ~/Documents/skill_3_quality_check

# 编辑配置文件
nano input/db_config.json
```

修改以下内容：

```json
{
    "name": "你的配置名称",
    "host": "你的 MySQL 地址",
    "port": 3306,
    "user": "你的用户名",
    "password": "你的密码",
    "database": "你的数据库名",
    "charset": "utf8mb4"
}
```

**注意事项**：
- `host` 不要包含 `jdbc:` 前缀
- `host` 不要包含端口号（端口号单独配置）
- 确保数据库账号有**查询权限**和**建表权限**

### 步骤 5：验证数据库连接

```bash
# 进入项目目录
cd ~/Documents/skill_3_quality_check

# 测试连接
python3 py_scripts/db_connector.py
```

### 步骤 6：执行评估

```bash
# 进入项目目录
cd ~/Documents/skill_3_quality_check

# 方式一：一键执行完整流程
python3 py_scripts/dq_quality_checker.py --all

# 方式二：分步执行
python3 py_scripts/dq_quality_checker.py --init    # 初始化环境（首次执行）
python3 py_scripts/dq_quality_checker.py --sync    # 同步元数据
python3 py_scripts/dq_quality_checker.py --check   # 执行质量检查
python3 py_scripts/dq_quality_checker.py --report  # 生成评估报告
```

## 五、常见问题排查

### 问题 1：pymysql 安装失败

**错误信息**：
```
ERROR: Could not find a version that satisfies the requirement pymysql
```

**解决方案**：
```bash
# 升级 pip
python3 -m pip install --upgrade pip

# 重新安装
pip3 install pymysql
```

---

### 问题 2：数据库连接失败

**错误信息**：
```
pymysql.err.OperationalError: (2003, "Can't connect to MySQL server...")
```

**排查步骤**：
1. 检查 `input/db_config.json` 中的 `host` 是否正确（不要有 `jdbc:` 前缀）
2. 检查数据库是否可达：`ping <host>`
3. 检查端口是否开放：`telnet <host> 3306`
4. 检查用户名密码是否正确

---

### 问题 3：权限不足

**错误信息**：
```
Access denied for user 'xxx'@'xxx'
```

**解决方案**：
联系 DBA 授予以下权限：
```sql
GRANT SELECT, CREATE, INSERT, UPDATE, DELETE, ALTER ON your_database.* TO 'your_user'@'%';
FLUSH PRIVILEGES;
```

---

### 问题 4：表结构缺失字段

**错误信息**：
```
Unknown column 'is_pk' in 'field list'
```

**解决方案**：
```bash
python3 py_scripts/dq_quality_checker.py --init
```

---

### 问题 5：规则解析失败

**错误信息**：
```
✅ 解析到 0 条规则
```

**解决方案**：
1. 检查 `references/rule_v2.md` 文件是否存在
2. 检查文件路径是否正确

## 六、执行时间估算

根据数仓表数量，执行时间大致如下：

| 表数量 | 预计时间 |
|--------|---------|
| 1-10 张 | 1-3 分钟 |
| 10-50 张 | 3-10 分钟 |
| 50-100 张 | 10-20 分钟 |
| 100+ 张 | 20+ 分钟 |

**优化建议**：
- 大表（>1000 万行）会自动使用 LIMIT 采样
- 可以分批次执行（先执行特定分层）
