# Kimball维度建模原则

## 维度建模概述

维度建模是一种数据仓库设计方法论，由Ralph Kimball提出。其核心理念是**以业务过程为中心**，通过事实表和维度表的组合，直观地表达业务数据关系。

## 维度建模四步法

### 第一步: 选择业务过程

业务过程是企业运营中发生的业务活动，如:
- 销售订单
- 库存变动
- 资金交易

**选择原则:**
- 选择最核心的业务过程
- 优先选择数据质量好的过程
- 考虑业务需求优先级

### 第二步: 声明粒度

粒度是事实表中单行数据的含义。

**粒度类型:**
| 类型 | 说明 | 示例 |
|-----|-----|-----|
| 事务粒度 | 每条记录代表一次业务动作 | 每笔订单 |
| 周期快照 | 定期记录状态 | 每日库存 |
| 累积快照 | 记录业务全过程 | 订单全流程 |

**声明原则:**
- 粒度越细越好
- 粒度必须明确
- 粒度决定维度

### 第三步: 确定维度

维度是描述业务过程环境的属性。

**常见维度:**
- 时间维度: 日期、时间
- 地理维度: 地区、门店
- 组织维度: 部门、公司
- 产品维度: 商品、品类
- 用户维度: 客户、会员

**维度设计原则:**
- 维度属性完整
- 维度描述性文字丰富
- 维度层次清晰

### 第四步: 确定事实

事实是业务过程的度量值。

**事实类型:**
| 类型 | 说明 | 示例 |
|-----|-----|-----|
| 可加事实 | 可跨维度聚合 | 销售金额 |
| 半可加事实 | 部分维度可加 | 库存数量 |
| 不可加事实 | 不可聚合 | 比率 |

**事实设计原则:**
- 使用数值类型
- 保留原始度量
- 避免过度计算

## 事实表设计

### 事实表分类

| 类型 | 特点 | 示例 |
|-----|-----|-----|
| 事务事实表 | 记录业务事件 | 订单明细 |
| 周期快照事实表 | 定期记录状态 | 每日库存 |
| 累积快照事实表 | 记录全流程 | 订单流转 |

### 事实表原则

1. **粒度一致性**: 同一事实表粒度相同
2. **维度一致性**: 关联维度含义一致
3. **事实完整性**: 包含完整度量
4. **键值唯一性**: 事实表有唯一标识

## 维度表设计

### 维度表结构

```
维度键 + 维度属性
```

**标准维度表结构:**
- 代理键: 系统生成的唯一标识
- 自然键: 源系统的业务标识
- 维度属性: 描述性字段
- 层次属性: 上级层次信息

### 维度表原则

1. **宽度适中**: 维度表可以较宽
2. **属性丰富**: 尽可能多的描述属性
3. **层次完整**: 包含完整的层次关系
4. **描述清晰**: 属性值有明确含义

## 缓慢变化维度(SCD)

### SCD类型

| 类型 | 处理方式 | 适用场景 |
|-----|---------|---------|
| SCD1 | 覆盖旧值 | 不需历史 |
| SCD2 | 新增记录 | 需要完整历史 |
| SCD3 | 增加字段 | 只需当前和上一版本 |

### SCD2实现

```sql
-- 维度表增加字段
effective_start_date DATE,  -- 生效开始日期
effective_end_date DATE,    -- 生效结束日期
is_current BOOLEAN          -- 是否当前版本
```

**处理规则:**
- 新记录: start_date=当前日期, end_date=9999-12-31
- 变更记录: 新增一条记录，旧记录end_date更新为当前日期

## ODS层与维度建模的关系

### ODS层定位

ODS层是维度建模的数据来源:
- 提供原始业务数据
- 保持数据一致性
- 支持数据回溯

### 关系映射

| ODS层 | 维度建模 |
|-----|---------|
| 用户表 | 用户维度 |
| 商品表 | 商品维度 |
| 订单表 | 订单事实 |
| 订单明细表 | 订单明细事实 |

### 最佳实践

1. ODS层保持源数据原貌
2. DWD层进行维度建模转换
3. 建立清晰的血缘关系
4. 保证数据一致性

## 维度建模检查清单

### 事实表检查

- [ ] 粒度是否明确
- [ ] 事实是否完整
- [ ] 维度外键是否正确
- [ ] 度量是否可加

### 维度表检查

- [ ] 代理键是否唯一
- [ ] 属性是否完整
- [ ] 层次是否清晰
- [ ] SCD处理是否正确

### 一致性检查

- [ ] 共享维度是否一致
- [ ] 事实粒度是否匹配
- [ ] 维度属性是否对齐
