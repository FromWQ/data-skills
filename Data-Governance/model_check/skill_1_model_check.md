# 模型规范检查技能

## 概述

模型规范检查用于对数据仓库各分层（ODS/DIM/DWD/DWS/ADS）的数据表进行质量评估，确保表结构符合数仓建设规范。

## 核心概念

### 数仓分层

- **ODS 层**: 原始数据层，保持源系统数据形态
- **DIM 层**: 维度表层，存储公共维度信息
- **DWD 层**: 明细数据层， cleaned 和标准化的明细数据
- **DWS 层**: 汇总数据层，按主题域聚合
- **ADS 层**: 应用数据层，面向应用场景

### 数据同步策略

| 策略 | 适用场景 | 增量标识 |
|-----|---------|---------|
| 日增量 (di) | 有增量标识字段，数据量大 | xxx_di |
| 日全量 (df) | 全量快照，数据量适中 | xxx_df |
| 小时增量 (hi) | 实时性要求高 | xxx_hi |

## 检查流程

### 步骤 1: 表命名检查

检查表命名是否符合 `{分层前缀}_{业务域}_{表名}_{增量标识}` 格式。

| 检查项 | 规范要求 | 错误示例 |
|-------|---------|---------|
| 命名格式 | 分层前缀 + 业务域 + 表名 + 增量标识 | user_info |
| 增量标识 | di/df/hi 后缀 | table_name |
| 表名长度 | ≤64 字符 | 过长表名 |

### 步骤 2: 字段规范检查

检查字段命名、注释、类型是否符合规范。

**必检项:**
- 字段命名使用小写字母 + 下划线
- 每个字段必须有注释，注释不能为空
- 数据类型选择合理，避免滥用 STRING

**类型选择原则:**
- ID 字段：使用 BIGINT 或 STRING
- 金额字段：使用 DECIMAL(p,s)，避免 DOUBLE
- 日期字段：使用 DATE 或 STRING(格式化)
- 短字符串：使用 VARCHAR(n)
- 长字符串：使用 STRING

### 步骤 3: 分区规范检查

检查分区设计是否合理。

| 增量类型 | 分区字段 | 分区格式 |
|---------|---------|---------|
| 日增量 (di) | dt | yyyyMMdd |
| 日全量 (df) | dt | yyyyMMdd |
| 小时增量 (hi) | dt, hh | yyyyMMdd, HH |

**检查要点:**
- 分区字段命名统一（使用 dt）
- 分区值格式正确
- 使用列式存储格式

### 步骤 4: 存储规范检查

检查存储格式和压缩配置。

```sql
-- 推荐配置示例
STORED AS ORC
TBLPROPERTIES ('orc.compress'='SNAPPY')
```

| 检查项 | 规范要求 |
|-------|---------|
| 文件格式 | ORC 或 Parquet |
| 压缩格式 | SNAPPY 或 ZLIB |

## 快速参考

### 表命名检查清单

- [ ] 表名包含分层前缀 (ods_/dwd_/dws_/dim_/ads_)
- [ ] 包含业务域标识
- [ ] 以增量标识结尾 (di/df/hi)
- [ ] 表名长度≤64 字符
- [ ] 全部小写，使用下划线分隔

### 字段检查清单

- [ ] 字段名小写 + 下划线
- [ ] 所有字段有注释
- [ ] 类型选择合理
- [ ] 同表类型风格一致

### 分区检查清单

- [ ] 分区字段命名为 dt
- [ ] 分区格式符合规范
- [ ] 使用列式存储

- [ ] 文件格式为 ORC/Parquet

## 输出约定

检查结果输出为两个文件：Excel 结果文件和 Markdown 报告文件。

### 输出路径与文件命名

| 配置项 | 规则 |
|-------|-----|
| Excel 结果 | output/model_check_result.xlsx |
| Markdown 报告 | output/model_check_report.md |

所有表的检查结果汇总输出到同一文件中。

### 评分规则

总分 100 分，评分项如下：

| 检查项 | 分值 | 说明 |
|-------|-----|------|
| 表命名检查 | 10 分 | 固定分值 |
| 字段规范检查 | 60 分 | 按字段注释、命名规范性计分 |
| 分区规范检查 | 10 分 | 固定分值 |
| 存储规范检查 | 20 分 | 存储格式、压缩配置 |

## 注意事项

1. **字段注释**：所有字段必须有 COMMENT 注释
2. **命名规范**：统一使用下划线命名，避免驼峰
3. **分区设计**：建议使用 dt 作为分区字段
4. **存储优化**：使用 ORC/Parquet + SNAPPY 压缩
5. **持续检查**：建议集成到 CI/CD 流程
